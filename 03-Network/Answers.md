
## רמה 1: יסודות רשת
1. **מה ההבדל העיקרי בין Layer 3 ל-Layer 4 במודל ה-OSI?**
Layer 3 (Network) אחראי על ניתוב (Routing) של חבילות מידע בין כתובות IP שונות ברשתות שונות. Layer 4 (Transport) אחראי על העברת נתונים בין תהליכים או יישומים שונים על גבי המארח באמצעות Ports, ומבטיח אמינות במידת הצורך.

2. **מה ההבדל בין TCP ל-UDP?**
TCP הוא פרוטוקול מבוסס חיבור (Connection-oriented) הדורש 3-Way Handshake ומבטיח שהמידע יגיע במלואו ובסדר הנכון. UDP הוא פרוטוקול ללא חיבור (Connectionless) שאינו מבטיח הגעה, אך הוא מהיר הרבה יותר ומתאים לסטרימינג או ל-VoIP.

3. **מה המשמעות של קידומת `/24` בכתובת IP?**
זהו סימון CIDR המציין ש-24 הביטים הראשונים בכתובת ה-IP מייצגים את הרשת, וה-8 הנותרים (מתוך 32) מיועדים למכשירים (Hosts). ברשת כזו יש 256 כתובות אפשריות.

4. **מה ההבדל בין כתובת IP ציבורית (Public) לפרטית (Private)?**
כתובת Public היא גלובלית וניתנת לניתוב ישירות באינטרנט. כתובת Private משמשת לשימוש פנימי בתוך רשת (כמו ב-VPC או LAN) ואינה ניתנת לניתוב באינטרנט, לכן נדרש NAT כדי לגשת החוצה.

5. **מהו Default Gateway?**
כתובת ה-IP של הנתב ברשת המקומית, שאליה נשלחת כל תעבורה המיועדת ליעדים מחוץ לרשת המקומית.

6. **מהו ARP וכיצד הוא מקשר בין Layer 2 ל-Layer 3?**
פרוטוקול ARP מתרגם כתובת IP (לוגית, Layer 3) לכתובת MAC (פיזית, Layer 2) כדי לאפשר למכשירים באותה רשת פיזית לתקשר זה עם זה.

7. **תאר את תהליך ה-TCP 3-Way Handshake.**
תהליך הקמת חיבור: הלקוח שולח SYN, השרת משיב ב-SYN-ACK, והלקוח מאשר ב-ACK.

8. **מהם Ephemeral Ports?**
פורטים המשמשים את הלקוח (Client) באופן זמני ליצירת חיבור יוצא לשרת. הם נבחרים באופן אקראי מטווח פורטים גבוה ומשתחררים בסיום החיבור.

9. **מהו MTU ומה קורה אם חבילת מידע גדולה ממנו?**
MTU מגדיר את גודל חבילת המידע המקסימלי שניתן להעביר (בדרך כלל 1500 bytes ב-Ethernet). אם חבילה גדולה ממנו, היא עוברת Fragmentation (פיצול), מה שעלול להאט את ביצועי הרשת.

10. **מה המטרה של Subnet Mask?**
לעזור למערכת ההפעלה או לנתב להפריד בין ה-Network ID ל-Host ID בכתובת ה-IP.

11. **מה ההבדל בין כתובת MAC לכתובת IP?**
כתובת MAC היא כתובת פיזית (Hardware) הצרובה על כרטיס הרשת (L2). כתובת IP היא כתובת לוגית (Logical) המוקצית על ידי הרשת (L3).

12. **מה ההבדל בין Switch ל-Router?**
Switch מחבר מכשירים בתוך אותה רשת (L2) לפי MAC. Router מקשר בין רשתות שונות (L3) לפי IP.

13. **מהו פרוטוקול ICMP והאם הוא משתמש ב-Ports?**
ICMP משמש להודעות שגיאה ודיאגנוסטיקה (כמו `ping`). הוא פועל ב-L3 ולכן **אין לו פורטים**.

14. **הסבר את תהליך ה-DHCP (DORA).**
התהליך הוא: Discover (לקוח מחפש שרת), Offer (שרת מציע כתובת), Request (לקוח מבקש את הכתובת), Acknowledge (שרת מאשר ושומר את הכתובת).

15. **מה החשיבות של כתובת ה-Loopback?**
זו כתובת וירטואלית (לרוב `127.0.0.1`) המאפשרת למחשב לתקשר עם עצמו לצורך בדיקות פנימיות של שירותים.

## רמה 2: שירותי רשת ופרוטוקולי אפליקציה
1. **ב-DNS, מה ההבדל בין רשומת A לרשומת CNAME?**
רשומת A מתרגמת דומיין ל-IP ספציפי. רשומת CNAME היא כינוי (Alias) המפנה דומיין אחד לדומיין אחר.

2. **מה ההבדל בין שגיאות HTTP מסוג 4xx ל-5xx?**
שגיאות 4xx (כמו `404`) מייצגות טעות של הלקוח (בקשה לא תקינה או חוסר הרשאות). שגיאות 5xx (כמו `500`) מייצגות כשל בצד השרת.

3. **מדוע SSL/TLS נחוץ אם הסיסמאות בבסיס הנתונים מוצפנות?**
הצפנת DB מגינה על נתונים במנוחה (Data at Rest). SSL/TLS מגן על הנתונים בזמן שהם עוברים ברשת (Data in Transit) מפני האזנות (Man-in-the-Middle).

4. **מהו CORS ומדוע נתקלים בו?**
זהו מנגנון אבטחה בדפדפן המונע מדומיין אחד לבצע בקשות ל-API בדומיין אחר ללא אישור מפורש מהשרת.

5. **מה ההבדל בין בקשת GET לבקשת POST?**
GET משמשת לשליפת מידע והיא Idempotent (לא משנה את השרת). POST משמשת לשליחת נתונים חדשים ומשנה את מצב השרת.

6. **מה ההבדל בין Reverse Proxy ל-Forward Proxy?**
Forward Proxy מסתיר את הלקוח (יושב בצד הלקוח). Reverse Proxy מסתיר את שרתי ה-Backend (יושב בצד השרת) ומבצע Load Balancing.

7. **האם HTTP הוא Stateful או Stateless? כיצד אתרים "זוכרים" אותנו?**
HTTP הוא Stateless (כל בקשה עצמאית). אתרים משתמשים ב-Cookies או Tokens (כמו JWT) כדי לשמור את מצב המשתמש (Session).

8. **הסבר את תהליך ה-DNS Resolution.**
הלקוח פונה ל-Resolver, שעובר דרך Root Server, משם ל-TLD, ומשם ל-Authoritative Name Server שמחזיר את ה-IP.

9. **מה ההבדל בין WebSocket לבקשות HTTP רגילות?**
HTTP עובד בשיטת Request-Response. WebSocket מאפשר ערוץ תקשורת דו-כיווני מתמשך (Full-Duplex) בזמן אמת.

10. **מה ההבדל בין קוד סטטוס 301 ל-302?**
301 הוא הפניה קבועה (Permanent). 302 הוא הפניה זמנית (Temporary).

11. **כיצד עובד אימות מבוסס SSH Key-Pair?**
שימוש בהצפנה אסימטרית: המשתמש מחזיק Private Key במחשב, וה-Public Key נמצא בשרת. השרת מאתגר את המשתמש, ורק ה-Private Key יכול לפצח את האתגר.

12. **מה ההבדל בין VPN ל-Proxy?**
VPN מצפין את כל התעבורה של מערכת ההפעלה (L3). Proxy מטפל בתעבורה של אפליקציה ספציפית בדרך כלל ללא הצפנה ברמת המנהרה.

13. **מהו SNI ב-TLS ומדוע הוא קריטי?**
SNI מאפשר לדפדפן לציין את הדומיין אליו הוא ניגש בתחילת ה-Handshake של TLS, מה שמאפשר להריץ מספר דומיינים עם תעודות SSL שונות על אותו כתובת IP.

14. **מהו CDN וכיצד הוא משפר ביצועים?**
רשת שרתים מבוזרת גיאוגרפית ששומרת Cache של תוכן סטטי. זה מקצר את המרחק הפיזי למשתמש ומוריד עומס מהשרת המרכזי.

15. **מהם היתרונות של HTTP/2 על פני HTTP/1.1?**
יתרון מרכזי הוא Multiplexing – העברת בקשות מרובות במקביל על גבי חיבור TCP אחד, בניגוד ל-HTTP/1.1 שהיה צריך לחכות לכל קובץ בנפרד.

## רמה 3: כלי מערכת ופתרון תקלות
1. **כיצד עובדת פקודת `traceroute`?**
היא שולחת חבילות עם TTL הולך וגדל. כל נתב בדרך מחסיר 1 מה-TTL, וכשהוא מגיע ל-0, הנתב מחזיר שגיאת "Time Exceeded", מה שמאפשר לזהות את כתובתו של כל נתב בדרך.

2. **איך בודקים אם שרת מרוחק מאזין בפורט 3306?**
משתמשים בפקודות כמו `telnet <IP> 3306` או `nc -vz <IP> 3306` (Netcat).

3. **איך בודקים ב-Linux איזה שירות מאזין בפורט 80?**
משתמשים בפקודה `netstat -tuln` או `ss -tuln`.

4. **איך בודקים רק את ה-HTTP Headers בעזרת `curl`?**
משתמשים ב-flag `-I` (למשל: `curl -I https://example.com`).

5. **איך בודקים לאיזה IP מתרגם דומיין דרך הטרמינל?**
בעזרת הפקודות `nslookup <domain>` או `dig <domain>`.

6. **מה ההבדל בין "Connection Refused" ל-"Connection Timeout"?**
"Connection Refused" אומר שהגענו ליעד אך הפורט סגור. "Connection Timeout" אומר שלא קיבלנו מענה כלל (בדרך כלל בגלל Firewall שחוסם/זורק חבילות).

7. **מה ההבדל בין הקבצים `/etc/hosts` ו-`/etc/resolv.conf`?**
`/etc/hosts` הוא קובץ מיפוי מקומי (Static) שהמערכת בודקת לפני DNS. `/etc/resolv.conf` מגדיר לאילו שרתי DNS חיצוניים לפנות.

8. **איך תופסים תעבורת רשת בפורט 80 בעזרת `tcpdump`?**
בעזרת הפקודה: `sudo tcpdump -i any port 80`.

9. **מהם `iptables` ב-Linux?**
כלי Firewall ברמת ה-Kernel המאפשר להגדיר חוקים של קבלה, דחייה או ניתוב של תעבורת רשת (Ingress/Egress).

10. **באילו פקודות משתמשים היום במקום `ifconfig`?**
משתמשים בחבילת `iproute2`, בעיקר ב-`ip addr` (לכתובות IP) ו-`ip route` (לניתוב).

11. **ping מצליח אך SSH נכשל ב-Timeout – מה הבעיה?**
בעיה בשכבת ה-Transport (L4). כנראה Firewall או Security Group חוסמים את פורט 22, או ששירות ה-SSH לא רץ.

12. **איך מודדים זמן תגובה של שרת באמצעות `curl`?**
בעזרת הפקודה `curl -o /dev/null -s -w "%{time_total}\n" <URL>`.

13. **מה עושה ההגדרה `net.ipv4.ip_forward=1`?**
היא מאפשרת למערכת ה-Linux להעביר חבילות בין ממשקי רשת שונים (Router functionality), קריטי לעבודה עם Docker ו-Kubernetes.

14. **מה תפקידו של `/etc/nsswitch.conf`?**
קובץ הקובע את סדר החיפוש של מידע במערכת (למשל, לחפש קודם בקובץ המקומי ורק אז ב-DNS).

15. **איך מוצאים שרתי דואר (MX) של דומיין?**
בעזרת הפקודה `dig <domain> MX`.

## רמה 4: רשתות בענן (AWS)
1. **מה ההבדל הארכיטקטוני בין Public Subnet ל-Private Subnet?**
ל-Public יש טבלת ניתוב שמובילה ל-Internet Gateway (IGW). ל-Private אין גישה ישירה לאינטרנט.

2. **איך מאפשרים לשרת ב-Private Subnet להוריד עדכונים?**
באמצעות NAT Gateway שמותקן ב-Public Subnet וניתוב התעבורה אליו מה-Private Subnet.

3. **מה ההבדל בין Security Group ל-Network ACL?**
Security Group הוא Stateful (ברמת ה-Instance). NACL הוא Stateless (ברמת ה-Subnet, דורש הגדרת Inbound ו-Outbound).

4. **מתי תבחר ב-ALB על פני NLB?**
ב-ALB (Layer 7) כשצריך ניתוב מבוסס Path או HTTP Headers. ב-NLB (Layer 4) כשצריך ביצועים גבוהים מאוד או Static IP.

5. **מה המשמעות של `0.0.0.0/0`?**
כל כתובת אפשרית באינטרנט הציבורי.

6. **איך ניגשים לשירותי AWS (כמו S3) ללא NAT Gateway?**
באמצעות VPC Endpoints, המאפשרים תקשורת פנימית על גבי הרשת של AWS.

7. **מה ההבדל בין VPC Peering ל-Transit Gateway?**
VPC Peering הוא חיבור נקודה-לנקודה בין שני VPCs. Transit Gateway הוא Hub מרכזי לניהול חיבורים בין מספר רב של VPCs.

8. **מה ההבדל בין Latency Routing ל-Weighted Routing ב-Route 53?**
Latency מפנה לשרת עם זמן התגובה המהיר ביותר למשתמש. Weighted מפנה תעבורה באחוזים מוגדרים.

9. **מה ההבדל בין Public IP אוטומטי ל-Elastic IP?**
Public IP אוטומטי משתנה ב-Reboot. Elastic IP הוא כתובת סטטית שנשמרת ל-Account שלך.

10. **מה תפקידו של Internet Gateway (IGW)?**
מאפשר תקשורת בין ה-VPC לאינטרנט ומבצע NAT בין כתובות פרטיות לציבוריות.

11. **מהו Bastion Host והיכן הוא ממוקם?**
שרת גישה מאובטח ב-Public Subnet שדרכו מתחברים למכונות בתוך ה-Private Subnet.

12. **מה ההבדל בין רשומת CNAME לרשומת ALIAS ב-Route 53?**
CNAME עובד לפי תקן DNS גלובלי. ALIAS הוא פתרון פנימי של AWS המאפשר להצביע ישירות על משאבי AWS (כמו ALB) ב-Root Domain.

13. **מה המטרה של Health Checks ב-Load Balancer?**
לוודא שהשרת מאחוריו תקין. אם הוא לא מגיב, ה-LB מפסיק לשלוח אליו תעבורה.

14. **מה ההבדל בין WAF ל-Security Group?**
WAF (Layer 7) מגן מפני התקפות אפליקטיביות (כמו SQL Injection). Security Group (L3/L4) מגן ברמת IP ופורטים.

15. **מה משמעות הסיומת `/32` ב-CIDR?**
מציין רשת המכילה כתובת IP אחת בלבד.

## רמה 5: רשתות בקונטיינרים ותזמור (Docker & Kubernetes)
1. **מה עושה ה-flag `-p` ב-`docker run`?**
מבצע Port Mapping בין ה-Host לפורט בתוך הקונטיינר.

2. **מה ההבדל בין `ClusterIP` ל-`NodePort` ב-Kubernetes?**
`ClusterIP` נגיש רק מתוך ה-Cluster. `NodePort` פותח פורט חיצוני על כל Node ב-Cluster.

3. **מדוע להשתמש ב-Ingress במקום ב-LoadBalancer לכל שירות?**
כדי לחסוך בעלויות של Load Balancers חיצוניים ולנהל ניתוב מרכזי מבוסס Path/Domain בתוך ה-Cluster.

4. **איך קונטיינרים באותו Host צריכים לתקשר ביניהם?**
באמצעות Custom Bridge Network שמאפשר Resolve לפי Container Name.

5. **מה תפקידו של `kube-proxy`?**
אחראי על תחזוקת חוקי ה-Network ברמת ה-Node (כמו `iptables`) כדי לנתב תעבורה ל-Service המתאים.

6. **איך עובד Service Discovery בעזרת CoreDNS?**
יוצר רשומת DNS אוטומטית לכל Service, כך שניתן לפנות אליו בשם ולא ב-IP.

7. **מהו Headless Service ומדוע הוא נחוץ ל-StatefulSets?**
שירות ללא ClusterIP. הוא מאפשר גישה ישירה ל-IP של כל Pod בנפרד, מה שקריטי לבסיסי נתונים במצב Stateful.

8. **איך מיישמים Network Policies ב-Kubernetes?**
על ידי הגדרת אובייקט `NetworkPolicy` המגדיר חוקי Ingress/Egress מבוססי Label.

9. **מהו CNI ב-Kubernetes?**
פלאגין שאחראי על הקצאת IP ל-Pods ועל קישוריות הרשת ביניהם.

10. **מדוע נהוג להוציא הגדרות Ingress ל-`values.yaml` ב-Helm?**
כדי להפוך את ה-Chart לדינמי ולהתאים את הדומיינים והגדרות הניתוב לסביבות שונות (Dev/Prod).

11. **מה המשמעות של הרצת קונטיינר עם `--network host`?**
הקונטיינר משתמש ישירות בממשק הרשת של ה-Host, ללא בידוד רשת.

12. **מדוע שינוי IP של Pod מחייב שימוש ב-Services?**
כי Pods הם אפימריים (מתים וקמים עם IP חדש). ה-Service מספק IP קבוע ו-DNS יציב.

13. **איך שירות מסוג `LoadBalancer` מקבל IP חיצוני בפועל?**
ה-Cloud Controller Manager ב-Kubernetes מבצע קריאת API לספק הענן (כמו AWS) ומקים Load Balancer חיצוני.

14. **מהן מדיניות "Egress" ב-Kubernetes?**
חוקי Firewall המגבילים לאילו יעדים Pods מורשים לשלוח תעבורה יוצאת.

15. **מהו Service Mesh ואילו בעיות הוא פותר?**
שכבת תשתית (כמו Istio) המנהלת תקשורת בין Microservices בעזרת Sidecars, ומספקת אבטחה (mTLS), ניטור, ו-Retry logic.
